## 使用方法

在文档主目录（`docsDirectory`）中创建一个名为 `index.json` 的文件，可以自定义文档的显示顺序。文件内容格式如下：

```json
[
  "README.md",
  {
    "title": "使用教程",
    "docs": [
      "使用教程/目录菜单排序须知.md",
      "使用教程/Markdown编写须知.md"
    ]
  },
  {
    "title": "语法支持示例",
    "docs": [
      "文档相关/Markdown语法样例.md",
      "文档相关/Mermaid图表样例.md"
    ]
  },
  "README_EN.md"
]
```

## 注意事项

1. **必须包含 `title` 和 `docs`**：每个文件夹对象都要有这两个字段，且 `docs` 里的文件路径必须与实际文件夹和文件名一致。

2. **只支持两层结构**：自定义排序仅允许两层（外层文件和内层文件夹），但默认排序支持多层结构。

3. **没有 `index.json` 时**：系统会按文件夹名称的字母顺序排序。

4. **`index.json` 格式错误时**：也会按默认排序处理。

5. **只构建列出的文件**：只有 `index.json` 里写明的文件才会被构建，未列出的文件不会显示。

6. **可调整文件位置**：通过修改 `docs` 中的路径，可以把文件从一个文件夹移到另一个文件夹，甚至放到最外层（例如直接写相对路径）。

7. **不能有重复的文件路径**：如果 `index.json` 里出现相同的文件路径，排序会失效，改用默认排序。
